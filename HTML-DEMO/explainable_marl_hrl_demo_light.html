01
Initialisation
Initialise environment \(\mathcal{E}\), state space \(\mathcal{S}\), action space \(\mathcal{A}\), \(N\) agents each with \(\pi_H\) and \(\pi_L\), action-value function \(Q(s,a)\), and explainability module \(\mathcal{X}\).
ENV
02
Episode Loop Begins
Reset \(\mathcal{E}\) and observe initial state \(s_0\). Loop continues until termination condition: task completion or maximum steps \(T\).
ENV
03
High-Level Policy Selects Macro-Action
For each agent \(i \in [N]\), the high-level policy selects a sub-goal: \(g \leftarrow \pi_H(s)\). This abstracts long-horizon planning into manageable coordination strategies.
High-Level
04
Low-Level Policy Executes Sub-Action
Given sub-goal \(g\), the low-level policy executes: \(a \leftarrow \pi_L(s, g)\). This concrete action (move, resource pick-up, task step) directly interacts with \(\mathcal{E}\).
Low-Level
05
Bellman Update & Policy Optimisation
Observe \((s', r)\) from \(\mathcal{E}\). Update \(Q\)-value: \[Q(s,a) \;\leftarrow\; Q(s,a) + \alpha\!\left[r + \gamma \max_{a'} Q(s',a') - Q(s,a)\right]\] Then update \(\pi_L\) and \(\pi_H\) via gradient descent.
Update
06
XAI Explanation Generated
Module \(\mathcal{X}\) produces explanation \(\mathcal{E}_i\) for agent \(i\)'s decision: identifies dominant features in \(s\), justifies \(\arg\max Q(s,a)\), and appends to the decision trace \(\{(s_t, a_t, r_t)\}_{t=0}^T\).
XAI
07
State Transition → Next Step
Set \(s \leftarrow s'\). Inner loop repeats for all agents. After 50 epochs, training accuracy reaches \(96.4\%\) and validation accuracy \(95.7\%\).
Iterate
Core Equations
Eq. 1 — Cumulative Reward
\[\displaystyle R = \sum_{t=0}^{T} r_t\]
Sum of all rewards \(r_t\) received by an agent over episode length \(T\)
Eq. 2 — Optimal Policy
\[\pi^*(a \mid s) = \arg\max_{a \in \mathcal{A}}\, Q(s, a)\]
Select action \(a\) maximising expected future reward \(Q(s,a)\) in state \(s\)
Eq. 3 — Bellman Update (Q-Learning)
\[Q(s,a) \leftarrow Q(s,a) + \alpha \!\left[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \right]\]
\(\alpha\) = learning rate, \(\gamma\) = discount factor, \((s',a')\) = next state-action pair
Eq. 4 — Hierarchical Composition
\[a = \pi_L\!\left(s,\; \pi_H(s)\right)\]
Low-level action \(a\) is conditioned on the sub-goal \(g = \pi_H(s)\) from the high-level policy
Pseudocode
INPUT: E (env), S (states), A (actions) OUTPUT: Optimised \(\pi_H, \pi_L\) + explanations Init E, N agents, πH, πL, Q(s,a), X FOR each episode: Reset E; observe initial state s WHILE not terminated: FOR each agent i ∈ N: g ← πH(s) // macro-action a ← πL(s, g) // sub-action Apply a to E s', r ← observe Update Q(s,a) via Bellman Update πL, πH Ei ← X.explain(s,a,r) // XAI Store trace + Ei s ← s' RETURN πH, πL, explanations

Why XAI Matters in Autonomous Systems
🚗
Autonomous Driving
Regulators require explanation for every decision taken by policy \(\pi(a \mid s)\). Black-box RL cannot be certified without interpretability.
🏥
Medical Systems
Clinicians must trace the causal path from state \(s\) to action \(a\) to \(r\) before trusting AI decisions in patient care.
🏭
Manufacturing Automation
Safety protocols require auditable \(\{(s_t, a_t)\}\) logs to trace coordination failures and prevent accidents.
🚁
Robotic Swarms
Human operators need to understand swarm coordination under \(\pi_H\) and intervene when emergent behaviours diverge from intent.
Research Gap Addressed
❌ Existing Gap
Most MARL and HRL models optimise \(\max_\pi \mathbb{E}[R]\) without any mechanism for interpretability. Performance-only frameworks cannot be deployed in safety-critical domains where accountability is non-negotiable.
✅ This Paper's Solution
Integrates module \(\mathcal{X}\) directly into the HRL loop — every \((s, a, r)\) tuple generates a human-readable explanation \(\mathcal{E}_i\), enabling debugging, certification, and user trust.
🔮 Future Work
Extension to intelligent transport networks, robotic swarms, deeper RL architectures, adaptive inter-agent communication protocols \(\mathcal{C}(i,j)\), and richer attribution-based XAI methods.
